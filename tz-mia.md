# Техническое задание: Интернет-магазин «Mia»

## 1. Общее описание проекта

**Название:** Mia — мультибрендовый бутик одежды  
**Язык сайта:** Русский  
**Тип:** Интернет-магазин (каталог + корзина + оформление заказа)  
**Целевая аудитория:** Гости отелей в Сочи, постоянные покупатели магазинов  
**Платформа:** Веб (desktop + мобильные устройства)  
**Количество товаров:** ~10–15 на старте (тестовые), до 200–500 в перспективе

### 1.1. Магазины (точки продаж)

Бизнес работает через три физические точки. Ассортимент может отличаться между магазинами:

| # | Название | Локация |
|---|----------|---------|
| 1 | Камелия | Сочи |
| 2 | Мантера | Сириус |
| 3 | Рэдиссон Лазурная | Сочи |

**Логика работы:** Пользователь при первом входе выбирает магазин (или видит предложение выбрать). Выбранный магазин сохраняется в сессии. Каталог показывает только товары, доступные в выбранном магазине. В шапке сайта всегда отображается текущий выбранный магазин с возможностью сменить. На карточке товара можно показать доступность в других магазинах текстом: «Также есть в: Камелия, Сочи».

## 2. Технологический стек

### 2.1. Frontend
- **Фреймворк:** Next.js 14+ (App Router) — React-фреймворк с серверным рендерингом, отличной SEO-оптимизацией и маршрутизацией
- **Язык:** TypeScript — строгая типизация, меньше багов, лучше автокомплит
- **Стили:** Tailwind CSS 4 — утилитарный CSS-фреймворк для быстрой и консистентной вёрстки
- **Анимации:** Framer Motion — плавные анимации, переходы страниц, hover-эффекты
- **Иконки:** Lucide React — минималистичные SVG-иконки
- **Шрифты:** Google Fonts — подберём пару: акцидентный (заголовки) + текстовый (body). Предлагаю: **Cormorant Garamond** (заголовки, лого) + **Manrope** или **Outfit** (body) — даёт ощущение бутика без вычурности

### 2.2. Backend (MVP)
- **API:** Next.js API Routes (встроенные) — для MVP не нужен отдельный бэкенд
- **База данных (перспектива):** PostgreSQL + Prisma ORM
- **Хранение данных на MVP:** JSON-файлы или Supabase (бесплатный tier) — быстрый старт без сложной инфраструктуры
- **Хранение изображений (перспектива):** Cloudinary или S3-совместимое хранилище

### 2.3. Админ-панель (MVP)
- Простая защищённая страница `/admin` с паролем
- Позже можно вынести в отдельное приложение или подключить CMS (Sanity, Strapi)

### 2.4. Деплой
- **Хостинг:** Vercel (бесплатный план для старта, идеально работает с Next.js)
- **Домен:** Подключается к Vercel за 5 минут

## 3. Дизайн-система

### 3.1. Общая стилистика

**Направление:** Refined minimalism / Luxury boutique  
**Референсы:** 12 Storeez (чистота, типографика, нейтральные тона), Kith (структура каталога, hover-эффекты), MyTheresa (карточка товара, аккордеоны), Farfetch (фильтры, компоновка каталога)

**Ключевые принципы:**
- Много белого пространства ("воздух") — контент дышит
- Акцент на фотографиях товаров — они главные герои
- Минимум декоративных элементов — никаких теней, градиентов, rounded-углов без необходимости
- Типографика как элемент дизайна — крупные заголовки, контрастные размеры
- Микроанимации для ощущения живости — но без перегруза

### 3.2. Цветовая палитра

| Токен | Цвет | Применение |
|-------|------|------------|
| `--bg-primary` | `#FFFFFF` | Основной фон |
| `--bg-secondary` | `#F5F3F0` | Фон карточек, секций (тёплый бежевый) |
| `--text-primary` | `#1A1A1A` | Основной текст |
| `--text-secondary` | `#6B6B6B` | Второстепенный текст, подписи |

### 5.1. Шапка сайта (Header) — присутствует на всех страницах

**Desktop-вид (1440px):**

```
┌──────────────────────────────────────────────────────────────────┐
│  ☰  Магазин: Камелия, Сочи ▾       MIA        🔍   ♡   🛒 (3) │
│                                                                  │
│      Женское    Мужское    Детское    Новинки    Бренды          │
└──────────────────────────────────────────────────────────────────┘
```

**Элементы:**
- **Бургер-меню (☰):** Открывает сайдбар слева (детали ниже)
- **Селектор магазина:** Текст «Камелия, Сочи ▾» — при клике dropdown с тремя магазинами. После выбора каталог обновляется
- **Лого «MIA»:** По центру. Шрифт Cormorant Garamond, 28px, weight 500. Клик → главная
- **Иконки справа:** Поиск (лупа), Избранное (сердце — на перспективу, пока неактивно), Корзина (сумка + счётчик товаров)
- **Навигация под лого:** Горизонтальная строка ссылок. Uppercase, Manrope 13px, tracking 0.1em. При hover — подчёркивание снизу с анимацией (slide-in)

**Mobile-вид (< 768px):**

```
┌──────────────────────────────────┐
│  ☰      MIA      🔍  🛒 (3)    │
│  📍 Камелия, Сочи ▾             │
└──────────────────────────────────┘
```

- Горизонтальная навигация скрывается, доступна через бургер
- Селектор магазина — отдельная строка под лого, мелким шрифтом
- Избранное скрыто на мобилке

**Поведение при скролле:**
- Header фиксированный (sticky)
- При скролле вниз — вторая строка (навигация) скрывается, header становится компактным (только лого + иконки)
- При скролле вверх — навигация плавно возвращается

### 5.3. Главная страница

Длинная страница-витрина, состоит из секций. Каждая секция — самостоятельный блок, который можно менять/добавлять через админку (в перспективе).

**Секция 1 — Hero-баннер:**
- Полноширинная фотография (или слайдер из 2–3 фото с автопрокруткой)
- Поверх фото: текст (например, «Новая коллекция Весна 2026» или «Mia — ваш стиль, ваш бутик»)
- Кнопка CTA: «Смотреть каталог»
- На MVP: плейсхолдер — красивая стоковая фотография с наложением
- Высота: 70vh (desktop), 50vh (mobile)

**Секция 2 — Категории:**
- Три карточки в ряд (desktop), стопкой (mobile)
- Каждая карточка — фото + название: «Женское», «Мужское», «Детское»
- При hover — лёгкий зум фотографии (scale 1.03, transition 500ms)
- Клик → переход в соответствующий раздел каталога

**Секция 3 — Новинки (карусель):**
- Заголовок: «Новинки»
- Горизонтальная прокрутка карточек товаров (6–8 штук)
- Стрелки навигации по бокам (desktop), свайп (mobile)
- Карточки — такие же, как в каталоге (описание ниже)
- Кнопка внизу: «Все новинки →»

**Секция 4 — Бренды:**
- Заголовок: «Наши бренды»
- Логотипы брендов в линию (если есть) или текстовый список в красивой типографике
- На MVP: просто названия брендов в ряд, стилизованные

**Секция 5 — О магазине (краткий блок):**
- Фото интерьера или фасада магазина + 2–3 предложения
- Кнопка: «Подробнее» → /contacts

**Секция 6 — Footer:**
- Логотип «Mia»
- Колонки: Каталог (ссылки), Информация (Доставка, Контакты), Соцсети (Instagram, Telegram)
- Копирайт: © 2026 Mia. Все права защищены.
- Адреса трёх магазинов

**URL:** `/catalog`, `/catalog/women`, `/catalog/men`, `/catalog/kids`, `/catalog/[category]`

**Layout:**

```

│  [Размер ▾] [Бренд ▾] [Цвет ▾] [Цена ▾]      Сортировка ▾    │
│                                                                 │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐              │
│  │         │ │         │ │         │ │         │              │

│  │         │ │         │ │         │ │         │              │
│  │         │ │         │ │         │ │         │              │
│  ├─────────┤ ├─────────┤ ├─────────┤ ├─────────┤              │
│  │ Бренд   │ │ Бренд   │ │ Бренд   │ │ Бренд   │              │
│  │ Название│ │ Название│ │ Название│ │ Название│              │
│  │ 12 500₽ │ │  8 900₽ │ │ 15 200₽ │ │  6 400₽ │              │

└─────────────────────────────────────────────────────────────────┘
```

**Карточка товара в каталоге:**
- **Фото:** Соотношение сторон 3:4. Объект-fit: cover. Скруглений нет (sharp edges)
- **Hover-эффект (desktop):** При наведении мыши на фото — изображение меняется на альтернативное. Если фотографий > 2, то позиция мыши по горизонтали определяет какое фото показать (как на 12 Storeez / Kith). Внизу фото появляются точки-индикаторы, показывающие текущее фото
- **Бренд:** Над названием, мелким шрифтом (11px, uppercase, muted color)
- **Название товара:** Manrope 14px, regular
- **Цена:** Manrope 15px, medium. Формат: `12 500 ₽`. Если есть скидка: зачёркнутая старая цена + новая цена акцентным цветом
- **Кружочки цветов:** Под ценой, если товар доступен в нескольких цветах. Маленькие цветные кружки (12px). При hover — tooltip с названием цвета. При клике — фото меняется на этот цвет
- **Бейдж «NEW»:** В левом верхнем углу фото, если товар новый. Accent color, uppercase, 10px

**Фильтры:**
- Строка фильтров над каталогом
- Каждый фильтр — dropdown:
  - **Размер:** XS, S, M, L, XL, XXL (чекбоксы, множественный выбор)
  - **Бренд:** Список брендов с чекбоксами
  - **Цвет:** Цветные кружки с названиями
  - **Цена:** Слайдер с двумя ползунками (от — до) + поля ввода
- **Сортировка:** По популярности, По цене ↑, По цене ↓, Новинки
- Активные фильтры отображаются как тэги под строкой фильтров с кнопкой ✕ для сброса каждого и «Сбросить все»
- На мобилке: фильтры скрываются за кнопку «Фильтры», открываются в нижней шторке (bottom sheet)

**Пагинация:**
- Кнопка «Показать ещё» (подгрузка по 20 товаров)
- Не infinite scroll (чтобы можно было добраться до footer)

**Галерея фото (левая часть):**

- Большое фото справа от миниатюр
- Клик на миниатюру → главное фото меняется (с плавным fade-transition)
- Клик на главное фото → открывается fullscreen-просмотр (lightbox) с зумом
- На мобилке: горизонтальная карусель фото (swipe), точки-индикаторы внизу

**Информация (правая часть):**
- **Бренд:** Ссылка на фильтр каталога по этому бренду, мелкий текст, uppercase
- **Название:** H1, Cormorant Garamond, 28px
- **Цена:** 18 500 ₽ — крупно, Manrope 22px, weight 500
- **Выбор цвета:** Кружочки цветов (20px). Активный — с бордером. При клике — галерея меняется на фото в этом цвете. Рядом текстом: «Цвет: Бежевый»
- **Выбор размера:** Кнопки-плитки (как на Kith). Доступные — кликабельные. Недоступные — серые, с cursor: not-allowed, полупрозрачные (opacity 0.4). Выбранный — чёрный фон, белый текст. Если размер не выбран и пользователь нажал «Добавить в корзину» — кнопки размеров подсвечиваются красным с сообщением «Выберите размер»
- **Кнопка «Добавить в корзину»:** Полноширинная, чёрный фон, белый текст, uppercase. При hover — лёгкое изменение (opacity 0.9 или bg-color shift). После добавления — текст меняется на «✓ Добавлено» на 2 секунды, затем возвращается. Одновременно: мини-уведомление справа вверху (toast) «Товар добавлен в корзину»
- **Избранное:** Текстовая ссылка с иконкой сердца. На MVP — визуально есть, но функционал ограничен (сохранение в localStorage)
- **Доступность в других магазинах:** Текстом под кнопкой покупки

**Аккордеоны (нижняя часть):**
- Описание (раскрыт по умолчанию)
- Состав и уход (свёрнут)

## 8. UX-взаимодействия и анимации

### 8.1. Каталог — hover на карточке товара

**Desktop (мышь):**
1. Фото карточки делится на N вертикальных зон (по количеству фотографий, максимум 5)
2. При наведении мыши — в зависимости от горизонтальной позиции курсора показывается соответствующее фото
3. Под фото появляются точки-индикаторы (маленькие чёрточки или точки), подсвечивающие текущее фото
4. Transition между фото: мгновенная смена (opacity swap, без slide)
5. При уходе мыши — возврат к первому фото

**Mobile (touch):**
- Эффекта при наведении нет (нет мыши)
- На карточке в каталоге — только первое фото
- При желании: под фото маленькие точки, свайп меняет фото (опционально, можно не делать для MVP)

### 8.2. Глобальные анимации

| Элемент | Анимация | Длительность |
|---------|----------|--------------|
| Сайдбар (меню, корзина) | Slide-in + overlay fade | 300ms ease-out |
| Dropdown фильтров | Fade-in + slide-down | 200ms |
| Аккордеоны | Height auto transition | 300ms ease |
| Добавление в корзину | Кнопка → «✓ Добавлено» + toast | 2000ms |
| Переход между страницами | Fade (через Next.js layout transitions) | 200ms |
| Hero-баннер (слайдер) | Crossfade | 600ms, auto: 5s |
| Карточка в каталоге (hover) | Лёгкий подъём тени (subtle) | 200ms |
| Кнопки (hover) | Opacity / background shift | 150ms |
| Изображения (загрузка) | Skeleton → fade-in | 300ms |

### 8.3. Skeleton-загрузки

Для всех динамических элементов — отображать серые пульсирующие плейсхолдеры (skeleton screens) до загрузки данных. Это создаёт ощущение быстроты.

## 11. MVP — план реализации по этапам

### Этап 1: Каркас и дизайн-система (3–5 дней)
- Настройка проекта (Next.js + TypeScript + Tailwind)
- Дизайн-токены (цвета, типографика, отступы)
- Компоненты: Button, Input, Badge, Icon, Skeleton
- Layout: Header, Footer, Sidebar
- Адаптивная сетка

### Этап 2: Каталог (5–7 дней)
- Страница каталога с сеткой карточек
- Карточка товара в каталоге (с hover-эффектом смены фото)
- Фильтры и сортировка
- Навигация по категориям (Women / Men / Kids)
- Селектор магазина
- Пагинация («Показать ещё»)

### Этап 3: Карточка товара (3–5 дней)
- Галерея фото (миниатюры + большое фото)
- Выбор цвета и размера
- Аккордеоны (описание, состав, доставка)
- Секция «Вам может понравиться»
- Breadcrumbs

### Этап 4: Корзина и чекаут (3–5 дней)
- Корзина (полная страница + мини-корзина сайдбар)
- Управление количеством, удаление
- Страница оформления заказа
- Форма данных клиента
- Способы оплаты (плейсхолдер + мессенджеры)
- Страница «Спасибо»

### Этап 5: Доп. страницы + Админка (3–5 дней)
- Контакты (с картой)
- Доставка и возврат
- Админ-панель: авторизация, список товаров, CRUD, управление заказами

### Этап 6: Контент и полировка (2–3 дня)
- Наполнение тестовыми товарами (10–15 позиций с placeholder-фото)
- Главная страница (hero, секции)
- Финальная проверка адаптивности
- Тестирование на мобильных устройствах
- Деплой на Vercel

**Итого MVP: ~20–30 дней** (при ежедневной работе)

- Интеграция с 1С (синхронизация остатков и цен)
- Реальный эквайринг (оплата картой через ЮKassa / Сбер / Тинькофф)
- Полноценный поиск по сайту (с автосаггестами)
- Wishlist / Избранное с сохранением между сессиями
- Личный кабинет пользователя
- История заказов для клиента
- Уведомления по SMS/Email о статусе заказа
- Промокоды и скидочная система
- Мультиязычность
- A/B тестирование
- Аналитика (Яндекс.Метрика, GTM) — хотя базовую можно добавить в MVP
- Интеграция с CRM
- Push-уведомления
- Программа лояльности

## 13. Открытые вопросы

1. **Название бренда vs название магазина:** Лого на сайте — «Mia». Бренды внутри (например, 12 Storeez, Max Mara) — это мультибрендовость. Нужен ли раздел «Бренды» с логотипами и описаниями, или достаточно просто фильтра?
2. **Фотографии:** На MVP — placeholder-изображения. Когда будут реальные фото — в каком формате и разрешении будут поставляться? Нужны ли гайдлайны для фотографа (белый/серый фон, определённый ракурс)?
3. **Уведомления о заказах:** На MVP — Telegram-бот для уведомления администратора? Или email? Или оба?
4. **Домен:** Есть ли уже домен? Если нет — нужно зарегистрировать (mia-store.ru, mia-boutique.ru и т.п.)
5. **Яндекс.Метрика / Google Analytics:** Подключать сразу в MVP?
