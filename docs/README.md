# Документация проекта

Эта папка хранит документацию, которая помогает:

- быстро понять архитектуру и структуру кода, не читая весь репозиторий;
- ускорить работу новых разработчиков и ИИ-агентов;
- фиксировать принятые решения (почему сделано так, а не иначе).

## Быстрые ссылки

- `AI_GUIDE.md` - "карта" проекта для ИИ: где что лежит, что является источником правды, какие инварианты.
- `ARCHITECTURE.md` - обзор архитектуры и основных потоков данных.
- `DATA_SOURCES.md` - режимы хранения данных: JSON vs Postgres (Prisma), переменные окружения, dual-write.
- `API.md` - краткое описание API, включая idempotency и админ-доступ.
- `DEVELOPMENT.md` - запуск локально, тесты, миграции/сидинг.
- `SCRIPTS.md` - утилиты импорта/нормализации/локализации медиа.
- `DECISIONS/` - ADR (Architecture Decision Records): решения и их контекст.

## Локальные документы (не коммитятся)

В репозитории допускаются локальные документы, которые не должны попадать в GitHub, например:

- заметки по парсерам/краулерам;
- cookie/токены/инструкции, привязанные к локальной машине;
- рабочие планы и черновики.

Папка: `docs-local/` (игнорируется через `.gitignore`).

## Принципы

- Документация должна быть "достаточно точной": описывать поведение кода, а не пожелания.
- Если изменили контракт/инвариант в коде, обновите соответствующий файл в `docs/`.
